一句话：Managed Agents 用 Session、Harness、Sandbox 三类稳定接口虚拟化 Agent——harness 可随模型换，执行环境 cattle 化，session 日志是 context window 外的 durable 真相源。

概览

Claude PlatformMeta-harnessLong-horizon

Claude Platform 上的托管长时程 Agent 服务。设计灵感来自操作系统：像 read() 不关心底层是 1970 年代磁盘还是 SSD 一样，Managed Agents 对 harness 实现、sandbox 形态保持接口稳定。

组件角色关键接口
SessionAppend-only 事件日志getEvents()
Harness（Brain）调用 Claude、路由 tool、context 工程wake, getSession, emitEvent
Sandbox（Hands）跑代码、改文件、外部执行execute(name, input), provision

§1 Harness 假设会过期 → 需要 meta-harness

Harness 常编码「模型做不到什么」的补偿逻辑。模型升级后，这些逻辑可能变成死重。

案例：context anxiety
  • Claude Sonnet 4.5 接近 context 上限时会过早收尾任务。
  • 团队在 harness 加了 context reset。
  • 换到 Opus 4.5：行为消失，reset 逻辑无用。

结论：不赌某一种 harness 长久有效，而是做 meta-harness——对 harness 本身做可替换抽象，让 Claude Code 或领域专用 harness 都能挂载。

§2 单容器耦合 = Pet（宠物）

早期 session、harness、sandbox 同处一容器：文件编辑是直接 syscall，边界简单。但带来三类问题：

  1. Pet 问题：容器挂 → session 丢；卡死需人工护理；WebSocket 流无法区分 harness bug、丢包、容器离线。
  2. 调试困境：只能 shell 进含用户数据的容器 → 实质上无法安全调试。
  3. 网络假设：资源必须在 harness 旁边；客户 VPC 只能对等网络或把 harness 搬到客户侧。

§3 解耦 brain / hands / session

Hands（Sandbox & Tools）

execute(name, input) → string
provision({resources})  // 容器死亡后重建

Harness 像调任意 tool 一样调容器。失败 → tool error → Claude 可 retry → 新 provision。

Harness（Brain）

wake(sessionId)
getSession(id)          // 读事件日志，续跑
emitEvent(id, event)    // 循环中持久化

Harness 本身也是 cattle：crash 后新实例 wake，从 session 恢复，无需进程内状态。

Session

getEvents()  // 按位置切片 interrogating 事件流

支持断点续读、rewind、重读某动作前上下文。

§4 安全边界：凭证进不了 sandbox

耦合风险：Claude 生成代码与平台 token 同容器 → prompt injection 读环境即可 spawn 无限制 session。缩小 token scope 仍是「假设 Claude 做不到 X」，会随模型变强失效。

结构修复：sandbox 内永远接触不到平台凭证。

场景模式
Git Init 时用 repo token clone，写入 local remote；sandbox 内 push/pull 可用，Agent 不持 token
MCP / OAuth Token 存 vault；专用 proxy 按 session 取凭证代调；harness 不知凭证

与 Claude Tag 的 Agent Proxy + credential store 同族范式。

§5 Session ≠ Claude 的 Context Window

长时程任务超出 context。Compaction、memory tool、trimming 都涉及不可逆的保留/丢弃——未来 turn 可能需要已删 token。

职责分离
  • Session：durable + interrogable 全量事件；只追加，不替 harness 做 compact 决策。
  • Harness：getEvents() 切片后任意变换再注入 Claude（含 prompt cache 布局、compact、trim）。

Session 是 context window 之外的 context object——类似 REPL 里 LLM 写代码 filter 外部对象，但存储在平台而非 sandbox 内。

§6 Many brains, many hands

Many brains → TTFT

Brain 在容器内时：每个 session 先付整容器成本（clone repo、拉事件），即使用户暂不需要 sandbox。TTFT（time-to-first-token）差。

解耦后：orchestration 读 pending events 即可开 inference；sandbox 懒 provision。

指标变化（作者声称）
p50 TTFT约 −60%
p95 TTFT> −90%

Many hands → 统一 execute

每个 hand：execute(name, input) → string。Harness 不知背后是容器、手机还是模拟器。Brain 之间可传递 hand。早期单容器是因为旧模型只能单 shell；模型变强后多环境路由成为能力而非负担。
