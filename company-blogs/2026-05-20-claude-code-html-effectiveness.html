一句话：Agent 越强，人类越可能不读计划就 hand off；HTML 用导航、SVG、交互把 spec 变成「愿意打开并参与」的交付物——Claude Code 团队几乎全面转向 HTML 作为人类可读输出。

概览

Claude Code输出格式Human-in-the-loop

Markdown 仍是 Agent 生态默认格式，但作者认为当任务变复杂时，Markdown 的信息带宽与可读性成为瓶颈：100 行以上难读完、ASCII 图替代真实可视化、分享 friction 高。HTML 单文件可离线打开，承载表格/CSS/SVG/JS，且与 Claude Code 的多源 context 摄入（FS、MCP、Chrome、git）结合，适合 spec、PR review、原型与 throwaway 编辑器。

§1 为何 HTML 比 Markdown 更合适

维度HTML 能力Markdown 常见退化
信息密度表格、CSS、SVG、canvas、嵌入代码ASCII 流程图、Unicode「估色」
可读性标签页、锚点导航、响应式布局长 wall of text，~100 行阅读上限
分享上传/发链接，浏览器即读附件或依赖 GitHub 渲染
交互slider、拖拽、live preview + export纯静态文本
ContextClaude Code 扫仓库生成带图报告同，但呈现层更弱
作者的核心动机（Stay in the loop）

格式是注意力工程：不是 vanity，而是对抗「Agent 做得越多，人读得越少」的 automation complacency。

§2 Markdown vs HTML：Learning 仓库视角

本文不主张 Learning 仓库全量替换 Markdown；而是把 HTML 作为Agent→人的富媒体通道，Markdown 保留为Agent↔Agent / 索引通道（见 format-guide.md）。

§3 六大用例（标签页）

Specs / 规划 / 探索

多 HTML 文件组成「web of specs」：brainstorm → mockup → 实现计划 → 新 session 带全部文件实现。

Generate 6 distinctly different onboarding approaches—vary layout,
tone, and density—in a single HTML grid. Label each with its tradeoff.

Code review / 理解

内联 diff 批注、严重性配色、模块流图——比 Markdown 更适合 PR 说明。

High：backpressure 未处理 slow consumer
Med：buffer 上限未文档化
Low：命名可更清晰

Design / 原型

Claude Design 基于 HTML；可 slider 调动画参数后 copy 到 React/Swift。

Prototype checkout button: play animation → turn purple.
HTML + sliders + copy parameters button.

Reports / 研究 / 学习

跨 Slack、代码库、git、网页综合；SVG explainer 单页。

Rate limiter HTML explainer: token-bucket diagram,
3–4 annotated snippets, gotchas section.

Custom editing UI

Throwaway 编辑器：拖拽 triage、feature flag 表单、prompt side-by-side。结尾必有 export。

  • copy as Markdown / JSON / prompt / diff
  • 单次任务专用，非产品化

验证 Agent

验证阶段让 verification agent 读取全部 HTML spec，对照实现——比单份 Markdown plan 上下文更完整。

§4 规划工作流
