# 博客摘录模板

适用于 `personal-blogs/` 与 `company-blogs/`。目标是 **提炼论点**，不是翻译全文。

```markdown
# 博客标题

> **作者**：
> **来源**：[原文链接](https://…)
> **发布**：YYYY-MM-DD
> **阅读日期**：YYYY-MM-DD
> **类型**：个人实践 / 公司 Research / Engineering

---

## 一句话

[作者核心主张，≤ 40 字]

## 为什么值得读

- 与主流认知的差异点（1–2 条）
- 与你当前学习主题的关联（如 Agent 循环、训练 infra）

## 核心论点

### 论点 1：[小标题]

- **作者说**：…
- **论据**：数据 / 案例 / 产品行为 / 引用论文
- **我的理解**：…（区分事实 vs 推断）

### 论点 2：…

（通常 2–4 个论点即可）

## 与已有知识的对照

| 主题 | 本文说法 | 其他来源（论文/笔记/产品） | 一致性 |
|------|----------|---------------------------|--------|
| 例：context cache | … | `opencode-note` §7 | 一致 / 补充 / 矛盾 |

- 矛盾处写明你更信哪边、为什么。

## 工程落点（公司 Blog 重点写）

- 产品或系统上 **实际做了什么**（可观察行为）
- **推断的实现手段**（标明「推断」）
- 对自建 Agent / 训练 pipeline 的启发

## 可行动清单

1. …
2. …
3. …

## 仍待验证

- [ ] 开放问题或需对照源码/论文的断言

## 关联阅读

- 论文：`papers/…`
- 应用笔记：`frontier-apps/…`
- 其他博客：…

---

*摘录完成：YYYY-MM-DD*
```

## 博客笔记写作要点

- **公司 Research Blog**：多写「发布了什么能力 → 可能架构 → 与竞品差异」。
- **个人博客**：多写「实践经验、踩坑、反直觉」；保留作者鲜明表述但压缩篇幅。
- 没有源码支撑时，避免写死实现细节；用「作者声称 / 合理推断」。
- 摘录 ≠ 转载；全文引用不超过原文 10%，且注明出处。
