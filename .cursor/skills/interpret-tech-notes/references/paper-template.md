# 论文笔记模板

```markdown
# 论文标题

> **作者 / 机构**：
> **链接**：arXiv / OpenReview / …
> **发表**：YYYY-MM
> **阅读日期**：YYYY-MM-DD
> **读者定位**：算法工程师，关注 Agent / LLM / 系统 …

---

## 目录

| 章节 | 主题 |
|------|------|
| [§1](#1-核心问题) | 核心问题 |
| [§2](#2-方法直觉) | 方法直觉 |
| … | … |

---

## 1. 核心问题

- 解决什么痛点？与已有工作的 gap 是什么？
- 问题形式化（输入 / 输出 / 优化目标）用一段话说明。

## 2. 方法直觉

用 **流程图或编号步骤** 说明方法，而非堆公式：

```mermaid
flowchart LR
    A[输入] --> B[关键模块]
    B --> C[训练/推理]
    C --> D[输出]
```

- **关键创新点**（1–3 条，每条一句话）
- **与最接近 baseline 的差异**

## 3. 实验与证据

| 实验 | 说明 | 结果要点 |
|------|------|----------|
| 主实验 | 数据集 / 指标 | 提升多少、是否显著 |
| 消融 | 去掉某组件 | 说明该组件必要性 |

- 作者声称的结论 vs 数据实际支持的结论（区分过誉）
- 复现难度：数据 / 算力 / 代码是否开源

## 4. 局限与开放问题

- 假设是否过强？
- 评估是否覆盖真实部署场景？
- 作者承认的局限 + 你看到的局限

## 5. 与 Agent / 工程实践的关联

> 本章最重要：写 **可迁移思想**，不是「这篇也用了 LLM」。

| 论文概念 | 工程对应 | 可参考项目/笔记 |
|----------|----------|-----------------|
| 例：long-horizon memory | context compact + 外部记忆 | Codex memories、`openclaw-note` §7 |

- 若做 Agent 系统，哪些设计可直接借鉴？哪些需要改造？
- 与 `frontier-apps/` 已有笔记的 connection（若有）

## 6. 个人评价

- **价值**：1–5 分 + 一句理由
- **是否值得精读全文 / 只读摘要 / 跳过**
- **后续动作**：要追的引用、要跑的代码、要与某篇博客对照读

---

*阅读完成：YYYY-MM-DD*
```

## 论文笔记写作要点

- Abstract 只能作引子；Method 要用自己的话重建逻辑。
- 图表编号要引用（「Figure 2 显示 …」）。
- 不复制大段原文；摘录不超过一句并加引号。
- 若只读摘要就写，必须在文首标注 **「仅基于摘要，待精读」**。
